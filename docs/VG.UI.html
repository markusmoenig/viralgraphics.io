(static) CallbackType

The different callback types which can be registered via VG.UI.Workspace.registerCallback and get called when the specified action is triggered inside the Visual Graphics model.
Properties:
Name Type Description
New callback Called when a New operation is triggered by the user.
UndoRedo callback Called when an Undo/Redo operation is triggered by the user. A single argument is passed to the callback with the undo/redo path. Only needed if the app needs some special UI updates for certain paths.
Open callback Called when the user selects open. This overrides the default Model behavior. Data to read into the app state is passed to the callback.
Save callback Called when the user selects save. This overrides the default Model behavior. Data to save needs to be returned by this callback.
LoggedStateChanged callback Called when the user logged state changes, i.e. logged in / logged out. Read out the arguments for details.
